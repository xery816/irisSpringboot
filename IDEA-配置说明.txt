IDEA运行配置说明
==================

问题：在IDEA中直接运行时找不到依赖的DLL

解决：配置Run Configuration

步骤：
----

1. 点击IDEA右上角的运行配置下拉框
   选择：Edit Configurations...

2. 找到或创建 IrisApplication 配置

3. 在 VM options 中添加：
   -Djava.library.path=F:\iris_springboot\native-libs\windows-x64

4. 在 Environment variables 中添加（重要！必须追加）：
   
   正确写法（追加到现有PATH）：
   PATH=$Path$;F:\iris_springboot\native-libs\windows-x64
   
   或者：
   PATH=%PATH%;F:\iris_springboot\native-libs\windows-x64
   
   ❌ 错误写法（会覆盖系统PATH）：
   PATH=F:\iris_springboot\native-libs\windows-x64

5. 点击 Apply 和 OK

6. 重新运行


详细配置：
--------

Run/Debug Configurations
├── Name: IrisApplication
├── Main class: cn.simbok.iris.IrisApplication
├── VM options: -Djava.library.path=F:\iris_springboot\native-libs\windows-x64
├── Environment variables: PATH=F:\iris_springboot\native-libs\windows-x64;%PATH%
└── Use classpath of module: iris-springboot


快捷方式：
--------

如果不想每次配置，使用脚本运行：
  build-and-run.bat  (已配置好PATH)


验证：
-----

运行后，日志应显示：
  Successfully loaded iris library on windows...

不应出现：
  Can't find dependent libraries
